Plain English

You already think in three states.

You just haven't built software that admits it yet.

Engineering

A traffic light with only two states causes accidents.

Red and green are not sufficient. Yellow encodes a third meaning: the intersection state is changing — do not commit to either decision yet. Every distributed system that has ever failed during a state transition was, in some sense, missing a yellow light. Ternlang makes yellow a first-class type.

Medicine

Clinical hold is not a refusal. It is a protocol state.

When a physician orders additional workup before reaching a diagnosis, that is not uncertainty — it is the correct decision given the data available. Binary AI systems are structurally unable to represent this: they must output a verdict. Trit = 0 gives the machine a formal right to request more evidence before committing to a recommendation.

Law

"Insufficient evidence" is a verdict, not an abstention.

A court that dismisses for lack of evidence has made a precise determination — not a failure to decide. This distinction exists in law, in medicine, and in engineering. It does not exist in binary logic. Ternlang gives it a computational name: trit = 0, a formal type, and runtime semantics that propagate it correctly through every operation.

Why does this matter for AI?

Current AI systems are forced to give you an answer. Even when they don't know. Even when the question is ambiguous. Even when two pieces of evidence directly contradict each other.

That's not intelligence — it's a coin flip dressed up as confidence.

Ternlang gives AI agents a mathematical right to say "not yet." Not as a refusal. As a routing instruction: I need more data before I can commit. Here's what I need. Give me that and I'll decide.

Binary system — forced verdict
"Utility signal is high. Proceeding with recommended action. Confidence: 91%."

"Utility: +0.9. Safety bounds: −0.9. Vectors in direct opposition. Aggregate scalar: 0.0. Execution suspended — awaiting updated parameters or human authorisation."

Rocket Launch

Three real outcomes: Go · Hold · No-Go

A range safety veto is absolute — if trit −1 fires from the safety axis, it overrides everything else and keeps the rocket on the pad. "Hold" is not a fault. It's a formal protocol state that says: sensors conflict, do not commit yet.

technical detail
Hard-block semantics: one hard dimension returning −1 terminates regardless of the aggregate gate score. The VM propagates hold through every downstream calculation — no silent suppression of dissenting signals. Compare to binary: the launch controller must pick Go or No-Go even when two sensors read different values.

Sensor Fusion

Camera says clear. Lidar says obstacle. Neither wins.

When sensors disagree, ternlang returns trit 0 — slow down and resolve the conflict before deciding. Consensus is earned, not forced. No sensor "wins" by default and discards the others.

technical detail
Four sensors fused: camera, lidar, radar, GPS. Each returns a trit. The aggregate uses a weighted consensus: unanimity → affirm/reject; single dissent → tend with confidence score attached. The confidence score tells the vehicle exactly how uncertain the system is, enabling graduated speed reduction rather than binary brake/no-brake.

Medical Triage

"Observe" is a real clinical outcome, not a bin for the indecisive

Unconscious patient = immediate intervention regardless of other vitals (hard veto). But there's a vast middle space between stable and critical that binary forces into one bucket or the other. Ternary gives "observe" its own routing and monitoring path.

technical detail
Vitals are scored individually as trits. The triage function matches on the unconscious flag first — a hard -1 gate that fires before any aggregate score is computed. The "observe" outcome (trit 0) routes to a separate nursing protocol with a 15-minute re-evaluation trigger, rather than being assigned to either the discharge queue or the critical-care ward.

Spam Filter

Quarantine is a first-class outcome, not a folder

Uncertain messages reach a human reviewer with a confidence score attached — they're not silently delivered or deleted. False positive rates drop because the filter is allowed to say "I don't know" instead of forced to guess.

technical detail
Trit 0 (quarantine) is a distinct execution path with its own side-effects: a reviewer notification is dispatched, the message is held in a separate queue, and a reason code is attached. Binary filters that have a "junk" folder are faking it — they still had to pick spam or not-spam first and then assign the folder as metadata. Ternary treats quarantine as a first-order type.

Hallucination Detector

LLMs hallucinate because they can't say "I need more data"

When a context window has low information density, this program emits trit 0 with a confidence score — a machine-readable "not enough evidence" signal. Any downstream agent can act on it: ask a follow-up, fetch a document, or route to a human.

technical detail
Information density is computed per-sentence using the same ternary compression algorithm used in MCP memory. Sentences scoring below 0.25 density contribute to a "sparsity mass" counter. When sparsity mass exceeds a threshold relative to total context size, the function emits trit 0 and attaches a breakdown: which sections were sparse, what keywords were missing. The LLM is never asked to generate from insufficient context.

RISC CPU Pipeline

Pipeline stall is not an error — it's trit 0

Translates Brandon Smith's Python 9-trit RISC simulator into native ternlang. In a CPU pipeline, "stall" (wait for a dependency) and "fault" (flush the stage) are naturally different states. Ternary models this without any special casing.

technical detail
Stall = trit 0: the fetch/decode stage suspends and holds its registers until the dependency is resolved, then resumes. Fault = trit -1: the stage flushes its register state and signals the interrupt controller. Affirm = trit +1: instruction proceeds to execute. The BET VM handles all three paths as native VM states — there's no "stall flag" bolted on top of a binary control path.
09_risc_fetch_decode.tern

Circuit Breaker

CLOSED · HALF-OPEN · OPEN — no workarounds needed

Every binary framework adds HALF-OPEN as a special-cased patch on top of a boolean state machine. In ternlang, it's simply trit 0 — the VM handles the transition natively. No extra flags, no edge-case code.

technical detail
CLOSED = affirm (+1): pass traffic normally. OPEN = reject (-1): fail fast, don't call downstream. HALF-OPEN = tend (0): let one test request through; if it succeeds, transition to CLOSED; if it fails, stay OPEN. The ternary match block handles all three cases in four lines of .tern — compare to the typical Go or Java implementation which requires a mutex-protected enum and a goroutine-based timer.

Loan Underwriting

The system knows when it shouldn't decide alone

Borderline loan applications route to a human underwriter via trit 0 — not as a rejection, but as an honest "model confidence is below the threshold for a binding financial decision." The system doesn't guess. It escalates.

technical detail
Five credit dimensions each return a trit (income, DTI, credit history, collateral, purpose). The aggregate gate requires unanimous affirm to auto-approve, single -1 from a hard dimension (e.g. active default) to auto-reject, and anything in between to emit trit 0 with a ranked list of the uncertain dimensions attached — giving the human underwriter a prioritised review queue rather than a raw application dump.
