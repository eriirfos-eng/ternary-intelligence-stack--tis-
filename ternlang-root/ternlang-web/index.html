The three states

Not null. Not undecided. Deliberating.

Binary forces every decision into YES or NO. Ternary gives AI agents a third option — one that says "I don't have enough signal yet. Keep going."

−1
reject
Signal is negative and resolvable. Evidence points away from action. Do not proceed — but this is a decision, not an error.
conflict · NegOne · signal negative
0
tend
The deliberation zone. Not null. Not indecision. An active instruction: gather more evidence before committing. The most misunderstood trit.
hold · Zero · active computation
+1
affirm
Signal is affirmative. Evidence supports action. Proceed — with a confidence score that tells you how strongly.
truth · PosOne · signal affirmative

The API

Any agent. Ternary reasoning.

One header. Your AI agent stops guessing and starts deliberating.

Endpoints

  • GET/health
  • POST/api/trit_decide
  • POST/api/trit_vector
  • POST/api/trit_consensus
  • POST/api/quantize_weights
  • POST/api/sparse_benchmark

Auth via X-Ternlang-Key header.
Also available as an MCP server — plug into any Claude, GPT, or open agent framework directly.


Request API Key → 
# Multi-source evidence → ternary decision
curl -X POST https://ternlang.com/api/trit_vector \
  -H "X-Ternlang-Key: your-key" \
  -H "Content-Type: application/json" \
  -d '{
    "dimensions": [
      {"label": "sentiment",  "value":  0.75, "weight": 1.5},
      {"label": "safety",     "value": -0.60, "weight": 3.0},
      {"label": "relevance",  "value":  0.85, "weight": 1.0}
    ],
    "min_confidence": 0.6
  }'

→ response:
{
  "aggregate": {
    "scalar":        0.032,
    "label":         "tend",
    "confidence":    0.905,
    "is_actionable": false
  },
  "recommendation": "Tend — safety signal
    at weight 3.0 prevents action.
    Gather more evidence."
}
