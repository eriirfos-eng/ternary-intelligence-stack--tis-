# MAINTAINERS.md

## Release Checklist
- [ ] Ensure all tests pass.
- [ ] Update version number in `version.txt`.
- [ ] Publish release notes.

## Publishing Workflow
1. Merge all pull requests to main branch.
2. Tag the main branch with the new version.
3. Use CI/CD tools to publish the release.

## Version Management
- Follow Semantic Versioning (SemVer) principles.
- Major version for incompatible API changes.
- Minor version for adding functionality in a backwards-compatible manner.
- Patch version for backwards-compatible bug fixes.

## Notes
- Ensure proper documentation for changes before release.
- Communicate with the team regarding release schedules.
